## Throughput math

The Windows scheduled task runs every 60s and processes all orchestrated projects in `~/.orchestra-goose/orchestrated-projects.txt`. With 6 projects x 89ms mean = **534ms total per cycle**. Headroom for ~100 projects per cycle if it ever scales. Negligible CPU load.

## Scope filter validation

Each project's briefing is correctly scoped to `project:<basename>`. wezbridge picks up its own L1 architecture claim (mm-90e3, score 0.677). Other projects don't see it — no cross-project bleed (the failure mode mm-c8d4~2 explicitly warned against). Validated: scope filter works.
